# 修正履歴

## 2024-XX-XX: tTransitionCellClass テーブル修正 ✅

**問題**: `tTransitionCellClass[512]` が単純な連番 (0x00-0x7F) になっており、
正しい等価クラスマッピングが行われていなかった。

**例**:
- Index 38: 旧 `0x26` → 正 `0x06` (クラス6)
- Index 41: 旧 `0x29` → 正 `0x05` (クラス5)
- Index 43: 旧 `0x2B` → 正 `0x0D` (クラス13)

**原因**: Transvoxelの等価クラステーブルは caseCode を 56 の等価クラスにマッピングするが、
テーブルが誤って0-127の連番として生成されていた。

**修正**: GitHub の公式実装 (https://github.com/EricLengyel/Transvoxel) から
正しいデータを取得し `TransvoxelTables.hlsli` を更新。

# どれのせいでおかしいのか判断するためのチェックリスト

## 1. Tableのデータがおかしい可能性 (TransvoxelTables.hlsli)

### 確認済み項目
- [x] `tTransitionVertexData` のデータ形式: 16ビット値（例: `0x2301`）
  - 下位4ビット: エンドポイントA (0-12)
  - 次の4ビット: エンドポイントB (0-12)
  - 上位8ビット: 再利用情報（現在未使用）
  - デコード: `idxA = vData & 0x0F`, `idxB = (vData >> 4) & 0x0F` → **正しい**

- [x] `transitionCellData[56]`: cellClass毎のジオメトリ情報
  - `geometryCounts`: 上位ニブル=頂点数、下位ニブル=三角形数
  - `vertexIndex[36]`: 三角形インデックス（最大12三角形×3頂点）

- [x] `tTransitionCellClass[512]`: caseCode → cellClass変換 **← 修正済み**
  - 下位7ビット: cellClass (0-55)
  - 最上位ビット: 反転フラグ (ワインディング反転必要)
  - **GitHub公式データで置換完了**

### 未確認項目
- [x] `transitionCellData` と `tTransitionVertexData` が同じソース（Transvoxel論文）から来ているか → 要確認
- [ ] 頂点インデックスの順序が反転フラグと整合しているか

## 2. 頂点生成の計算がおかしい可能性

### caseCode生成 (サンプリング)
```hlsl
if (d < voxelTerrainInfo.isoLevel) caseCode |= (1u << i);
```
**疑問点**: 
- Transvoxelでは通常 `d >= isoLevel` で solid (ビット=1)
- 現コードは `d < isoLevel` でビットセット → 等値面が反転している可能性
- 三角形の表裏だけでなく、形状自体が変わる可能性あり

### 座標変換
```
Canonical座標系 (kTransitionCornerOffsets)
    ↓ GetMappedDensity: Canonical → Physical (サンプリング用)
    ↓ 頂点生成: Canonical → Physical (同じ変換を適用)
```

**疑問点**:
- サンプリング時の変換:
  ```hlsl
  mappedOffset[axis] = localOffset[kPermutation[dirIndex][axis]]
  ```
- 頂点生成時の変換:
  ```hlsl
  mappedPos[axis] = physicalPos[kPermutation[dirIndex][axis]]
  ```
- これらが本当に「同じ変換」であるべきなのか、それとも逆変換が必要なのか要確認

### kTransitionCornerOffsets の配置
```
High-res面 (0-8): X=0 平面上の 3×3 グリッド
  0-----1-----2
  |     |     |
  3-----4-----5
  |     |     |
  6-----7-----8

Low-res面 (9-12): X=1 平面上の 2×2 グリッド
  9----------10
  |           |
  |           |
  11---------12
```

## 3. その他の可能性

### 偶数セル間引きロジック
```hlsl
if ((dirIndex == 0 || dirIndex == 1) && (cellCoord.y % 2 != 0 || cellCoord.z % 2 != 0)) 
    isEvenCell = false;
```
- Transvoxel では LOD 境界面で 2×2 セルを 1 つのトランジションセルに統合
- 奇数セルをスキップするロジックが正しいか確認必要

### worldOrigin の計算
```hlsl
float3 worldOrigin = worldPosBase - 0.5 * float3(step);
```
- MCの中心基準と合わせるためのオフセット
- Transvoxel のサンプリング基準点と一致しているか確認必要
